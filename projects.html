🛡 Behaviour-Based Endpoint Detection & Response Lab

Designed and implemented a behaviour-based Endpoint Detection and Response (EDR) environment using Wazuh and Sysmon to simulate enterprise security platforms such as CrowdStrike Falcon Complete.

The lab demonstrates how modern SOC environments collect endpoint telemetry, detect attacker behaviour, and automatically respond to security incidents.

Lab Architecture
  • Windows Endpoint (DC1) with Wazuh Agent and Sysmon
  • Wazuh Manager for detection and event correlation
  • Host-only virtual network for attack simulation
Endpoint Telemetry
  • Windows Security Event Logs monitoring
  • Sysmon process creation and command line monitoring
  • Network connection telemetry
  • Parent-child process relationship tracking
Behaviour-Based Detection Engineering
  • Developed custom detection rules in Wazuh
  • Detected encoded PowerShell execution
  • Correlated suspicious process and command activity
  • Identified multi-stage attack patterns
Automated Incident Response
  • Implemented Wazuh Active Response automation
  • Automatically isolated compromised endpoint
  • Disabled network connectivity via containment scripts
  • Terminated malicious command shells
SOC Alerting
  • Integrated Telegram Bot API for SOC notifications
  • Alerts include host information and containment status
Attack Simulation
  • Developed PowerShell script to simulate attacker behaviour
  • Generated encoded PowerShell commands
  • Executed system reconnaissance commands
  • Simulated outbound network connections
MITRE ATT&CK Mapping
  • T1059.001 – PowerShell Execution
  • T1027 – Obfuscated / Encoded Commands
  • T1082 – System Discovery
  • T1071 – Application Layer Protocol
